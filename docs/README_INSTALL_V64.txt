СОЛНЦАНЕТ CRM — v64 AUTO SAVE FIX

Проблема со скриншота:
admin.js отправляет в NocoDB поля "Направление", "Авто", "Авто услуги", но после сохранения в ответе/базе они остаются пустыми. Поэтому старый admin.js падает с ошибкой:
"NocoDB сохранил не все данные..."

Что исправляет v64:
1. Скрывает и очищает для направления "Авто":
   - Итоговый м² / Итоговый м2
   - Адрес
   - общий верхний Материал / Плёнка
2. Материал остается только внутри строк услуг авто.
3. Перед сохранением чистит авто-payload: не отправляет общий материал и адрес.
4. Обновляет Cloudflare Function /update-zayavka:
   - делает PATCH в NocoDB;
   - перечитывает запись после сохранения;
   - повторяет проверку, если NocoDB отдал старые данные;
   - дает понятную ошибку, если в таблице NocoDB не созданы нужные колонки.

КРИТИЧНО: в таблице заявок NocoDB должны существовать колонки:
- Направление       тип: Single line text / SingleSelect
- Авто              тип: Single line text / Long text
- Авто услуги       тип: Long text
- Общая стоимость   тип: Number / Decimal / Single line text

Колонка "Пленка" теперь не обязательна для авто, потому что материал пишется в каждой строке услуги.

Установка:
1. Загрузить файлы из папки functions/ в папку functions/ проекта Cloudflare Pages:
   - _nocodb.js
   - update-zayavka.js

2. Загрузить файл:
   assets/admin-v64-auto-clean-save-fix.js
   в папку assets/ проекта.

3. В admin.html подключить ПОСЛЕ основного admin.js и после прошлых v63-скриптов:

<script src="assets/admin-v64-auto-clean-save-fix.js?v=64"></script>

4. Сделать deploy в Cloudflare Pages.

5. После деплоя открыть сайт с очисткой кеша:
   Ctrl + F5
   или добавить к адресу:
   ?v=64

Проверка:
1. Открыть заявку.
2. Выбрать направление "Авто".
3. Убедиться, что "Итоговый м²", "Адрес" и общий "Материал/Плёнка" исчезли.
4. Добавить услуги авто, материал и сумму в строках услуг.
5. Сохранить.
6. В NocoDB проверить, что заполнены: Направление, Авто, Авто услуги, Общая стоимость.

Если после v64 появится ошибка о колонках — нужно добавить эти колонки в NocoDB. Без колонок браузер физически не сможет сохранить эти данные в базу.
