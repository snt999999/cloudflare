# СОЛНЦАНЕТ — сайт и админ-панель v7

Готовый комплект для Cloudflare Pages + GitHub.

## Что изменено в v7

1. Публичная форма на главной странице теперь реально отправляет заявку.
   - Форма вызывает Cloudflare Function `/create-public-lead`.
   - Заявка сохраняется в NocoDB.
   - Если задана переменная `WEB3FORMS_ACCESS_KEY`, копия заявки отправляется на почту через Web3Forms.

2. На форме убрана логика «Отправить в Telegram».
   - Основная кнопка теперь: `Отправить заявку`.
   - Telegram, MAX и телефон оставлены как дополнительные способы связи.

3. В форму добавлены поля:
   - услуга;
   - желаемая дата;
   - желаемое время;
   - адрес;
   - информация об объекте;
   - комментарий.

4. Добавлена защита от простого спама:
   - скрытое honeypot-поле;
   - проверка имени и телефона;
   - защита от мгновенной автозаполненной отправки.

5. Админ-панель обновлена:
   - редактирование даты, времени, услуги, адреса;
   - отмена заявки через «корзину отмен»;
   - восстановление заявки из корзины;
   - отдельный раздел «История изменений»;
   - история внутри карточки заявки;
   - исправлена дата `Сегодня` под Екатеринбург.

6. Отчёт по зарплатам стал рабочим:
   - ставки за м² по каждому монтажнику;
   - доплаты;
   - авансы / удержания;
   - выбор периода;
   - выбор статусов;
   - выбор монтажников;
   - режим деления м² поровну или полный объём каждому;
   - предварительный расчёт в админке;
   - экспорт CSV для Excel.

7. Cal.com webhook восстановлен и теперь не пустой.
   - Принимает POST от Cal.com.
   - Создаёт заявку в NocoDB.
   - Если находит существующий `Cal Booking ID`, обновляет заявку, а не создаёт дубль.
   - Можно защитить секретом `CAL_WEBHOOK_SECRET`.

## Переменные Cloudflare Pages

В Cloudflare Pages откройте:

`Workers & Pages → ваш проект → Settings → Variables`

Нужные переменные:

```text
ADMIN_PASSWORD=ваш_пароль_админки
NOCODB_TOKEN=ваш_токен_NocoDB
NOCODB_ENDPOINT=https://app.nocodb.com/api/v3/data/ptvxn8nmuwc08y3/mgp2zjsuv4id5tp/records
WEB3FORMS_ACCESS_KEY=ключ_из_Web3Forms
```

`WEB3FORMS_ACCESS_KEY` можно добавить позже. Без него заявка будет сохраняться в NocoDB, но письмо на почту не уйдёт.

Дополнительно для Cal.com:

```text
CAL_WEBHOOK_SECRET=любой_секретный_ключ
```

Если `CAL_WEBHOOK_SECRET` задан, в webhook Cal.com нужно передавать этот секрет в query-параметре:

```text
/cal-nocodb-v5?secret=ваш_секретный_ключ
```

## Важное по NocoDB

Основные поля, которые используются сейчас:

```text
Имя клиента
Телефон
Услуга
Дата записи
Время записи
Адрес
м2
Итоговый м2
Комментарий клиента
Комментарий администратора
Статус
Монтажники
Ответственный
Cal Booking ID
```

Для общей истории изменений на всех устройствах желательно добавить в таблицу NocoDB дополнительные текстовые поля:

```text
История изменений
Дата отмены
Причина отмены
```

Если эти поля не добавлены, сайт всё равно будет работать: основные изменения сохранятся, а история будет храниться локально в браузере администратора.

## Как загрузить

1. Распаковать архив.
2. Залить файлы в GitHub-репозиторий проекта.
3. Cloudflare Pages автоматически соберёт новую версию.
4. В Cloudflare добавить переменную `WEB3FORMS_ACCESS_KEY`.
5. Проверить `/test`, `/nocodb-test` и отправку формы с главной страницы.
