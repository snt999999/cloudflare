СОЛНЦАНЕТ CRM v62 — последняя версия по прошлому чату
Дата сборки: 09.07.2026

Главная цель v62:
чтобы данные в NocoDB действительно добавлялись, изменялись и удалялись без старых дублей, зависаний и ошибки ThrottlerException: Too Many Requests.

Что добавлено и исправлено:

1. Все операции NocoDB через Cloudflare Functions
- сайт больше не должен напрямую долбить NocoDB из браузера;
- добавлены серверные функции list-records, create-record, update-record, delete-record, batch-records;
- токен NocoDB остается только в Cloudflare Environment Variables;
- все запросы идут последовательно, с повтором при 429 / Too Many Requests.

2. Защита от ThrottlerException
- добавлен retry с паузой, exponential backoff и учетом Retry-After;
- пакетные операции выполняются по очереди, не параллельно;
- при ошибке сохранения карточка НЕ закрывается и пользователь видит понятное сообщение.

3. Надежное редактирование заявок
- сохранение отключает кнопку до завершения операции;
- локальный список обновляется только после успешного ответа NocoDB;
- добавлен IdempotencyKey, чтобы не плодить дубли при повторном клике;
- добавлена нормальная обработка ошибок.

4. Безопасное удаление
- физическое удаление заменено на корзину: статус «Удалена», поля DeletedAt и Причина удаления;
- удаленные/отказные заявки скрываются из активных списков, календаря и зарплаты;
- старые записи остаются в NocoDB для истории и восстановления.

5. Исправление проблемы Google Calendar
- если запись была перенесена, потом удалена, повторный перенос больше не должен упираться в старый локальный флаг «уже перемещено»;
- при удалении очищаются локальные признаки переноса/скрытия для этой заявки и Google event id;
- событие можно заново импортировать/создать после удаления старой карточки.

6. Карточка больше не закрывается при выделении текста мышью
- отключено закрытие dialog по клику на фон;
- закрытие только через кнопки «Сохранить», «Отмена», «Закрыть», Escape.

7. Авто-отчет по зарплате
- добавлен отдельный экспорт авто-отчета с суммами и расчетом ЗП;
- расчет берет направление «Авто», авто, пленку, услуги и общую стоимость;
- ставки вынесены в AUTO_SERVICE_RATES, их можно поправить в одном месте;
- отчет не должен быть пустым, если авто-заявки реально есть в NocoDB.

8. Service Worker
- кеш поднят до solncanet-v62;
- API-запросы Cloudflare Functions не кешируются.
